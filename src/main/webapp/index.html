Let your users guide your development by building your views first and separating the backend completely

  • Build your views first after a round of user-centric design: Frontend developers can build views without any backend knowledge
  • Treat your views as mockups when needed: all views can be valid HTML5 documents
  • Transform your views with real data: use CSS-like syntax to transform HTML mockups into real documents

AJAX support so simple it'll make your head spin

  • Turn regular forms into AJAX forms with two changes
  • Load slow parts of a page after the page is sent to the user with a single change
  • Easily all back to the server on any event

Reason about real-time applications without worrying about threading complexities

  • Send real-time updates as messages to actors
  • Put any number of real-time components on any page
  • Advanced connection management handles browser connection limits transparently
  • Uses server-side continuations to support thousands of connections without tying up threads unnecessarily

Let Lift take care of some of the most common security problems on the web for you

  • Type-safe URLs let you fail URLs before broken data reaches your application
  • Trivial access control based on simple functions
  • No views or files are accessible unless you explicitly expose them
  • HTML is always escaped before reaching the user, so you never have to worry about script tag injection in Lift views*
  • Opaque identifiers for form fields means resistance to replay and CSRF attacks

*—Note that scripts can also be injected via attributes. Always avoid binding user-specified data to event handling attributes.
